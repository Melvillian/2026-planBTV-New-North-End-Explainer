# Simplify Build via Bun --filter Dependency Ordering — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the custom `gen:build-order` build-ordering machinery with Bun's native `--filter` topological ordering, by upgrading Bun to the latest stable (1.3.14) and setting `"build": "bun --filter '*' build"`.

**Architecture:** Bun ≥1.3.9's default `bun --filter '*' build` builds workspace packages in dependency order (a dependent waits for its dependency) while parallelizing independent packages — making the computed build order unnecessary. NOTE: `--sequential` is explicitly the WRONG flag (Bun docs: "`--parallel` and `--sequential` do not respect dependency order"); the simplification relies on the DEFAULT `--filter` behavior, not `--sequential`.

**Tech Stack:** Bun workspaces, TypeScript, vitest, Docker, Render, GitHub Actions.

**Background:** On the repo's pinned bun@1.3.3, plain `bun --filter '*' build` was reproduced to FAIL with `TS2307: Cannot find module '@melvillian/common-lib'` (dependents build before `common-lib` finishes — [oven-sh/bun#12203](https://github.com/oven-sh/bun/issues/12203)). Bun's v1.3.9 [blog](https://bun.com/blog/bun-v1.3.9) and [--filter docs](https://bun.com/docs/pm/filter) document `--filter` as respecting dependency order. The gate in Task 1 verifies the upgrade actually fixes this on THIS repo.

**Decisions (from the user):**

- Pin Bun to **latest stable (1.3.14)**.
- **If the Task 1 gate fails** (clean `bun --filter '*' build` still mis-orders on the upgraded Bun): STOP, revert the build-script change, keep `gen:build-order`, and report. Do not proceed to later tasks.

## Task 1: Upgrade Bun + flip build script + VERIFY ordering (GO/NO-GO GATE)

**Files:**

- Modify: `package.json` (`packageManager`, `build`)
- Modify: `.bun-version`
- Modify: `bun.lock` (regenerated)

This task is the gate. It does NOT yet delete `gen:build-order` (that's Task 2) — so if the gate fails we can stop cleanly.

- [ ] **Step 1: Upgrade Bun to latest stable**

Run: `bun upgrade`
Then: `bun --version`
Expected: `1.3.14` (or newer). Record the actual version `X.Y.Z` — use it verbatim everywhere a version is pinned below. If it is **below 1.3.9**, STOP and report (the upgrade didn't take); do not continue.

- [ ] **Step 2: Pin the Bun version in package.json and .bun-version**

In `package.json`, change:

```jsonc
  "packageManager": "bun@1.3.3",
```

to (using the version from Step 1):

```jsonc
  "packageManager": "bun@1.3.14",
```

Overwrite `.bun-version` with exactly:

```
1.3.14
```

(single line, the version from Step 1).

- [ ] **Step 3: Flip the build script to native --filter ordering**

In `package.json`, change the `build` script from the long generated chain:

```jsonc
  "build": "bun --filter @melvillian/common-lib build && bun --filter api build && bun --filter @melvillian/claude-api build && bun --filter @melvillian/openai-summarizer build && bun --filter cli build",
```

to:

```jsonc
  "build": "bun --filter '*' build",
```

Leave the `gen:build-order` script in place for now (removed in Task 2).

- [ ] **Step 4: Reinstall so the lockfile matches the new Bun**

Run: `bun install`
Expected: success; `bun.lock` may update. (No source changes.)

- [ ] **Step 5: THE GATE — verify a clean build orders correctly**

Run: `bun run clean && bun run build`
Expected: SUCCEEDS with NO `TS2307: Cannot find module '@melvillian/common-lib'`. The build output should show `@melvillian/common-lib` completing before `@melvillian/claude-api` and `@melvillian/openai-summarizer` (a dependent waits for its dependency).

Run it **twice more** to rule out a lucky race:

```bash
bun run clean && bun run build
bun run clean && bun run build
```

Expected: succeeds all three times.

- [ ] **Step 6: GATE DECISION**

- **If all three clean builds SUCCEEDED:** the native ordering works. Proceed.
- **If ANY clean build FAILED with TS2307 (or wrong order):** the upgrade did NOT fix ordering. STOP. Revert the `build` script change in `package.json` back to the generated chain from Step 3 (keep the Bun version bump or revert it too — report both options), do NOT proceed to Tasks 2–4, and report: "Bun X.Y.Z still mis-orders `--filter` builds; keeping gen:build-order." This ends the plan.

- [ ] **Step 7: Verify the rest of the suite still passes**

Run: `bun run typecheck && bun run test && bun run lint:check`
Expected: all pass. (`gen:build-order` test still exists and passes here — it's removed in Task 2.)

- [ ] **Step 8: Commit**

```bash
git add package.json .bun-version bun.lock
git commit -m "build: use native bun --filter dependency ordering; upgrade bun to 1.3.14"
```
